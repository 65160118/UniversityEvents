# โครงการ : ระบบจัดการกิจกรรมมหาวิทยาลัย (Event Management System)

## 1. ชื่อโครงการ & ภาพรวม

### ชื่อโครงการ
ระบบจัดการกิจกรรมมหาวิทยาลัย (Event Management System)

### ภาพรวมโครงการ
โครงการนี้มีวัตถุประสงค์เพื่อพัฒนาระบบสารสนเทศสำหรับบริหารจัดการกิจกรรมภายในมหาวิทยาลัย โดยช่วยให้ผู้จัดกิจกรรมสามารถสร้าง แก้ไข และควบคุมจำนวนผู้เข้าร่วมกิจกรรมได้อย่างเป็นระบบ ในขณะเดียวกัน นักศึกษาและผู้เข้าร่วมสามารถสมัครและเช็กชื่อเข้าร่วมกิจกรรมผ่านระบบออนไลน์ได้อย่างสะดวก ลดขั้นตอนการจัดการเอกสาร และเพิ่มความถูกต้องของข้อมูลกิจกรรม

---

## 2. วัตถุประสงค์ & เกณฑ์ความสำเร็จ

### วัตถุประสงค์
1. เพื่อพัฒนาระบบสำหรับสร้างและจัดการกิจกรรมของมหาวิทยาลัย
2. เพื่อรองรับการสมัครเข้าร่วมกิจกรรมผ่านระบบออนไลน์
3. เพื่อบันทึกและตรวจสอบการเข้าร่วมกิจกรรมด้วยระบบเช็กชื่อ
4. เพื่อควบคุมและจำกัดจำนวนผู้เข้าร่วมกิจกรรม
5. เพื่อแสดงรายงานและสถิติของกิจกรรมในรูปแบบ Dashboard

### เกณฑ์ความสำเร็จ (Success Criteria)
- ผู้ใช้งานสามารถสร้าง สมัคร และเช็กชื่อกิจกรรมได้ครบทุกฟังก์ชัน
- ระบบสามารถจำกัดจำนวนผู้เข้าร่วมได้ถูกต้อง 100%
- Dashboard แสดงข้อมูลกิจกรรมและสถิติได้ถูกต้องและเข้าใจง่าย
- ระบบสามารถใช้งานได้จริงโดยไม่ซับซ้อน

## 4. ตารางเวลา & Milestones
ระยะเวลาโครงการ: 15 สัปดาห์

### Phase 1: วิเคราะห์และวางแผน (สัปดาห์ที่ 1–5)
- รวบรวมและจัดทำ Requirement ของระบบ
- วิเคราะห์ความเสี่ยงเบื้องต้นของโครงการ
- จัดทำเอกสาร Project Charter , SRS และ User Stories  
  **Milestone (D1):** Project Charter และ Requirements Document สำเร็จ

### Phase 2: ออกแบบระบบ (สัปดาห์ที่ 6–7)
- ออกแบบโครงสร้างระบบ (System Architecture)
- ออกแบบฐานข้อมูล (Database Schema)
- กำหนดแนวทางการพัฒนาระบบและโครงสร้างโค้ด  
  **Milestone (D2):** System Design Document และ Database Design สำเร็จ

### Phase 3: พัฒนาระบบและทดสอบเบื้องต้น (สัปดาห์ที่ 8–10)
- พัฒนาฟังก์ชันสร้างและจัดการกิจกรรม
- พัฒนาฟังก์ชันสมัครเข้าร่วมกิจกรรม
- พัฒนาฟังก์ชันเช็กชื่อเข้าร่วมกิจกรรม
- ทดสอบการทำงานของระบบในระดับฟังก์ชัน (Functional Testing)  
  **Milestone (D3):** ระบบหลักสามารถใช้งานได้ครบตาม Requirement

### Phase 4: ทดสอบและปรับปรุงคุณภาพระบบ (สัปดาห์ที่ 11–13)
- ทดสอบระบบโดยรวม (System Testing)
- ตรวจสอบความถูกต้องของข้อมูลและกระบวนการทำงาน
- แก้ไขข้อผิดพลาดและปรับปรุงประสิทธิภาพระบบ
- ตรวจสอบคุณภาพและความปลอดภัยของระบบเบื้องต้น  
  **Milestone (D4):** ระบบผ่านการทดสอบและมีคุณภาพพร้อมใช้งาน

### Phase 5: สรุปผลและนำเสนอผลงาน (สัปดาห์ที่ 14–15)
- พัฒนา Dashboard แสดงผลข้อมูลกิจกรรมและสถิติ
- จัดทำเอกสารสรุปโครงการและเอกสารประกอบการนำเสนอ
- เตรียมระบบสำหรับการสาธิต (Demo)
- นำเสนอผลงานโครงการ  
  **Milestone (D5):** โครงการเสร็จสมบูรณ์และพร้อมนำเสนอ

## 6. ผู้มีส่วนเกี่ยวข้อง & การสื่อสาร

### ผู้มีส่วนเกี่ยวข้องหลัก
- นักศึกษา (ผู้เข้าร่วมกิจกรรม)
- อาจารย์ / ผู้จัดกิจกรรม
- ผู้ดูแลระบบ
- ทีมพัฒนาโครงการ

### แผนการสื่อสาร
- ประชุมทีม: สัปดาห์ละ 1-2 ครั้ง
- รายงานความคืบหน้า: หลังจบแต่ละ Milestone
- การแก้ปัญหา (Escalation): Developer → QA → Scrum Master → Product Owner
